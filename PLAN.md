# EduAI Campus 开发计划

## 1. 计划目标

本计划基于已确认的首期范围制定：B 端学校/教育机构版，终端覆盖 Web + 微信小程序，目标学生为初高中生。

本计划用于指导后续工程构建，当前阶段仅形成开发计划，不实施代码开发。待本计划确认后，才能进入项目初始化、编码、测试和交付。

一期 MVP 周期确认为 2 周，目标是优先打通主线闭环，确保产品可行；二期再围绕精细化伴学、多端体验、复杂风控和运营能力扩展。

## 2. 首期交付范围

### 2.1 业务范围

- 学校/机构租户管理
- 教师、学生、家长账号体系
- 班级、课程、学生关系管理
- 初高中课程知识库管理
- AI 伴学智能体
- AI 知识库对话
- 学生 Web / 小程序学习入口
- 家长小程序协同入口
- 教师/机构 Web 管理后台
- 权限透明干预
- 基础风控合规
- 学习数据可视化
- 成绩趋势图表、教师点评、家长奖励兑现记录
- 教师录入成绩和 Excel 模板导入成绩
- 主修学科：数学、物理、化学、英语
- 非学科方向：计算机应用、Java 开发、Python 开发等
- 支持自定义学科分类和科目

### 2.2 不纳入首期

- 原生 iOS / Android App
- 完整直播课堂
- 教务排课系统
- 复杂 LMS 系统
- 支付订阅和计费系统
- 开放插件市场
- 高精度作文自动评分

## 3. 推荐技术方案

### 3.1 前端

- Web 管理端：Vue 3 + TypeScript + Vite + Vue Router + Pinia
- UI 组件：Element Plus
- 数据可视化：ECharts
- 小程序：uni-app + TypeScript
- 请求层：统一封装 API Client、Token 刷新、错误处理

选择理由：

- Vue 3 与小程序生态兼容性较好。
- Web 管理端和小程序可复用部分 TypeScript 类型、接口定义和业务规则。
- Element Plus 适合 B 端管理后台快速落地。

### 3.2 后端

首选方案：

- Java Spring Boot 3
- Spring Security + JWT
- MyBatis Plus 或 JPA
- PostgreSQL
- Redis
- pgvector 或 Qdrant

备选方案：

- Python FastAPI

建议首期采用 Java Spring Boot，原因是 B 端权限、审计、组织架构、后台管理和长期维护更适合强类型工程结构。

### 3.3 AI 与知识库

- AI 伴学：围绕单个学生建立长期伴学智能体，支持画像、评估、规划、提醒和阶段总结；推送提醒默认关闭，开启后按配置规则执行
- 大模型网关：封装模型供应商调用，预留多模型切换能力
- RAG：文档解析、切片、向量化、召回、重排、生成
- 知识库隔离：按租户、课程、年级、学科隔离
- 安全策略：输入前检、输出后检、敏感风险记录

### 3.4 部署

- 本地开发：Docker Compose
- 生产部署：Nginx + 后端服务 + PostgreSQL + Redis + 向量库
- 日志：结构化日志 + 操作审计日志
- 监控：服务健康检查、接口耗时、AI 调用耗时、错误率

## 11. 验收标准

首期完成时必须满足：

- 学校/机构管理员可完成组织、班级、课程、账号管理。
- 教师可维护课程知识库并查看班级学习数据。
- 学生可通过 Web 或小程序进行 AI 知识库问答。
- 每个学生可拥有 AI 伴学智能体，并获得学习评估、规划和阶段总结。
- AI 伴学推送提醒默认关闭，开启后严格按照设置规则推送。
- AI 伴学长期记忆默认保留 2 周，可在 90 天上限内自定义。
- 家长可通过小程序查看孩子学习概览和风险提醒。
- 教师可查看学生各科成绩变化趋势，进行点评并给出薄弱部分指导建议。
- 教师可手动录入成绩，也可使用标准 Excel 模板导入成绩。
- 家长可查看孩子各科成绩变化趋势，进行评价，并记录物质奖励或精神奖励的品级与兑现状态。
- 家长和教师可按权限查看 AI 伴学摘要，不能越权查看敏感细节。
- 权限限制和干预原因对学生透明可见。
- 风险内容能被拦截、分级、记录和复核。
- 数据看板能展示学生、班级、机构维度的核心指标。
- README.md 能说明背景、架构、数据库、功能、部署和运维。

## 12. 执行风险

### 12.1 小程序与 Web 双端成本风险

Web 管理端和小程序端同时建设会增加前端工作量。建议首期小程序聚焦学生和家长核心流程，复杂管理能力只放 Web。

### 12.2 AI 成本与响应速度风险

知识库问答涉及向量检索和模型调用，成本和响应速度不稳定。需要模型分层、缓存和限额策略。

### 12.3 B 端租户隔离风险

学校/机构数据必须严格隔离。数据库查询、文件存储、向量索引都必须带租户边界。

### 12.4 未成年人合规风险

初高中学生仍涉及未成年人数据保护。需要最小化采集、授权留痕、敏感数据加密和数据删除机制。

### 12.5 风控误判风险

教育问答可能被误判为敏感内容。需要风险分级、人工复核和反馈纠错机制。

### 12.6 AI 伴学过度依赖风险

AI 伴学具有长期陪伴属性，容易让学生过度依赖 AI 或把 AI 当成权威判断来源。需要限制连续高频使用、强制启发式答疑、关键风险协同教师/家长，并明确 AI 不替代教师、家长和专业心理支持。

### 12.7 奖励兑现争议风险

家长奖励属于家庭激励行为，系统首期只记录规则、达成情况、奖励类型、奖励品级和兑现状态，不直接接入支付，也不强制家长兑现。需要在页面文案中明确奖励由家长自行承诺和执行，避免平台承担资金或履约责任。

## 13. 两周 MVP 实施节奏

### 第 1 周

- 第 1 天：工程初始化，完成后端、Web 管理端、小程序基础框架。
- 第 2 天：完成租户、用户、角色、班级、课程基础模型。
- 第 3 天：完成登录鉴权、教师/学生/家长基础权限。
- 第 4 天：完成学生 AI 伴学智能体基础数据结构、默认 2 周记忆策略、90 天上限配置。
- 第 5 天：完成 AI 问答主流程、伴学今日建议、推送默认关闭配置。

### 第 2 周

- 第 6 天：完成教师成绩录入、Excel 导入模板和导入批次记录。
- 第 7 天：完成学生与家长成绩趋势图表。
- 第 8 天：完成教师点评与薄弱点指导建议。
- 第 9 天：完成家长评价、物质/精神奖励规则、奖励品级和兑现状态记录。
- 第 10 天：完成主线联调、基础测试、README 与部署说明草稿。

MVP 验收重点：

- 学校/机构可建立班级、课程、学生、教师、家长关系。
- 学校/机构可维护主修学科、非学科方向和自定义科目。
- 学生可使用 AI 问答和 AI 伴学基础建议。
- 伴学推送默认关闭，开启后按规则触发。
- 长期记忆默认 2 周，配置上限不能超过 90 天。
- 教师可录入和导入成绩，可查看趋势并点评。
- 家长可查看趋势，可评价并记录奖励兑现。
- 风控和权限边界可追溯。

## 14. 待确认事项

- 知识库资料来源是平台预置、机构上传，还是两者都支持。
- 首期接入的大模型供应商。
- 是否需要教师端独立小程序，还是首期只做教师 Web。
- Excel 成绩导入模板字段是否需要兼容现有学校格式。
- 家长奖励品级枚举是否由平台固定，还是允许家长自定义。
