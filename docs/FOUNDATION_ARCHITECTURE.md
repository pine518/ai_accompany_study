# EduAI 基座与业务模块架构

## 1. 目标

基于 Forge Admin 复用通用企业后台能力，同时保证教育业务不依赖 Forge 插件实现细节，后续可独立测试、替换和演进。

## 2. 模块边界

```text
forge-admin-server
  ├── forge-plugin-system
  ├── forge-plugin-message
  ├── forge-plugin-ai
  └── eduai-business

eduai-business
  └── forge-starter-web
```

依赖方向固定为：

```text
启动装配层 -> 基座插件
启动装配层 -> 教育业务
教育业务 -> 基座公开 starter/API
```

禁止：

```text
教育业务 -> Forge 插件实现
Forge 基座 -> 教育业务
小程序 -> 基座数据库
```

## 3. 基座模块

基座负责：

- 登录认证、Token 和客户端管理。
- 租户、组织、用户、角色、菜单和数据权限。
- 操作日志、登录日志、在线用户和系统监控。
- 文件、Excel、消息、配置、缓存和 ID 能力。
- AI 供应商、模型、通用 Agent、流式对话和会话存储。

基座不负责：

- 学生画像、班级、学科和家校关系。
- 成绩、点评、奖励和伴学计划。
- 教育知识库 RAG 和未成年人风控策略。

## 4. 教育业务模块

`backend/eduai-business` 当前承载：

- 学科与自定义分类。
- 学生 MVP 档案。
- 成绩趋势、成绩录入和导入模板。
- 教师点评。
- 家长物质/精神奖励记录。
- AI 伴学摘要和推送/记忆设置。

当前 API 使用内存仓库保持 MVP 演示兼容，MySQL 表结构已在 `eduai_mvp.sql` 建立。下一阶段将逐个用 Repository 替换内存实现，Controller 契约保持稳定。

## 5. 解耦机制

- Maven：`eduai-business` 不声明任何 `forge-plugin-*` 依赖。
- 包约束：教育业务统一位于 `com.mdframe.forge.eduai..`。
- ArchUnit：测试禁止教育业务依赖 `com.mdframe.forge.plugin..`。
- 数据：教育表统一使用 `edu_` 前缀，系统表保持 Forge 原命名。
- 接口：管理端和小程序只调用 HTTP API，不直接使用基座内部对象。
- AI：后续通过 `AiGateway` 端口对接 Forge AI 插件，领域层不直接调用插件 Service。

## 6. 安全基线

- 学生 AI 会话读取和删除必须校验当前用户。
- 管理员审计接口与学生接口分离。
- 租户边界覆盖 MySQL、Redis、文件、向量库、消息和任务。
- 主动推送默认关闭。
- 长期记忆默认 14 天，配置范围 1-90 天。
- 教师和家长只读取授权摘要，不直接读取学生完整对话。
- 生产环境密钥通过环境变量注入。

## 7. 测试策略

- 架构测试：业务模块不得依赖插件实现。
- 单元测试：伴学默认值、记忆上限、成绩规则和奖励状态。
- 集成测试：认证、租户隔离、角色权限和数据库仓库。
- API 测试：学生、教师、家长越权场景。
- 前端测试：动态菜单、权限页面和核心表单。
- 小程序测试：登录绑定、角色切换和伴学主流程。

## 8. 上游升级

1. 读取 `backend/UPSTREAM.md` 的基线提交。
2. 获取上游变更并仅选择需要的 commit。
3. 优先审查认证、租户、AI 会话和依赖版本变化。
4. 运行后端测试、管理端构建和小程序构建。
5. 更新基线提交和变更记录。

禁止直接用新上游目录覆盖 `backend`。
